# Stitch Portfolio Brief

Use `DESIGN.md` as the source of truth for the design system and use the uploaded screenshots as visual reference for the current implementation.

## What this project is

Design a **personal portfolio website** for **Mohammed Hisham Haris**, a **UI/UX Designer & Full-Stack Web Developer** based in **Kannur, Kerala**.

This is not a generic landing page or product site. It is a **freelance-oriented portfolio** that should help him:

- present himself as both a designer and developer
- attract website clients
- communicate taste, professionalism, and technical capability
- show selected work and future writing
- make it easy for clients to contact him

## What the website should feel like

The mood should be:

- clean
- cinematic
- editorial
- premium
- calm
- dark but readable
- professional and client-facing

Avoid:

- startup landing page cliches
- bright gradients
- neon or cyberpunk styling
- cartoon illustrations
- generic SaaS dashboard aesthetics
- playful or noisy visual language

## Core structure

Keep the site as a **single-page tabbed portfolio** with a **persistent sidebar on desktop**.

Main sections:

1. About
2. Resume
3. Services
4. Portfolio
5. Blog
6. Contact

## Key content and positioning

Name:

## Layout requirements

- persistent sidebar on desktop
- sidebar stacks above content on smaller screens
- tabbed workspace layout, not a long scrolling landing page
- refined card-based content sections
- responsive mobile behavior with horizontally scrollable navigation if needed
- clear visual separation between sidebar and main workspace

## Visual system

Follow the design system in `DESIGN.md`, especially:

- deep charcoal page background
- warm gold accent
- soft panel borders
- restrained blur and shadow
- elegant serif display typography
- modern sans-serif UI text
- premium, quiet, architectural surfaces

Recommended overall palette:

- near-black and charcoal base
- warm off-white text
- muted gold accents
- subtle plum, teal, and muted red only in project-card art areas

## Section guidance

### About

- strong headline
- short positioning statement
- clear explanation of value
- highlight cards
- process/workflow summary

### Resume

- experience timeline
- education or growth timeline
- tools/skills groups
- placeholders are acceptable, but they should look intentional

### Services

- make this section client-facing
- show 3 core service categories
- include short deliverables
- explain ideal client fit

### Portfolio

- show 4 selected project or case-study cards
- keep them polished even if they use placeholder content
- each should feel like curated work, not a generic gallery tile

### Blog

- keep room for future writing
- show article preview cards with category, title, summary, and reading time

### Contact

- direct contact methods
- premium dark contact form
- trustworthy and conversion-oriented

## Contact form

The implemented site already supports a real Netlify-compatible contact form. Keep the UI looking premium and trustworthy.

Visible fields:

- Name
- Email
- Project type
- Message
